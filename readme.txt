      Опис проекту Pershokursniki.jar
+++++++++++++++++++++++++++++++++++++++++++++
Структурно проект складається із 4 класів:
Student, Prof, Group, Main;

В проекті реалізована можливість створювати 
екземпляри класу Group. Конструктор має такі
параметри:
String name - ім'я групи;
int size - розмір групи(кількість студентів);

Список деяких методів:
addStudents(); //додає студентів
addProfessor(); //додає професора
addChiefOfGroup(); //вибирає старосту

getChiefName(); //виводить ім'я старости
getChiefFeatures(); //виводить особисті якості

gotoLesson(); //запускає перекличку

Після створення об'єкту Group, за допомогою
методу addStudents() створюється відповідно
n об'єктів students, що є екземплярами
Student[], де n = size.
За допомогою методу addProfessor() додається
професор.
Староста групи вибирається на основі суми
особистих показників кожного із студентів:
береться максимальна сума показників.
Метод gotoLesson() запускає перевірку журналу
за допомогою методу checkStudentsOf(Group group)
із класу Prof. Відсутність студентів реалізується
випадковим чином в методі 
Group.makeAbsentStudents()

Клас Prof має наступні поля:

name; //ім'я професора
group; //група професора
age; //вік професора

За замовчуванням за допомогою Group.addProfessor() в 
групі створюється професор із
name=DEFAULT_PROF_NAME;
що можна змінити за допомогою Prof.setName()
для group встановлюється значення Group.name
Вік професора встановлюється випадковим чином 
від 18 до 87 років, що має вплив на стиль мови 
професора.

деякі методи класу Prof:

checkStudentsOf(); //перевіряє присутність групи
sayHello(); // виводить привітання

Клас Student має такі поля:

ACHIVEMENTS; //досягнення
RESPECT; //повага
STAMINA; //витривалість
SEX; //стать
chief; //староста(true|false)
group; //ім'я групи
absent; // відсутність (true|false)

всі константи характеристик вибираються для
кожного студента випадковим чином - від 0 до
10 балів, стать вибирається випадково.
Поле chief вказує чи є даний студент старостою
за замовчуванням має значення false
поле absent вказує на присутність студента
за замовчуванням має значення false

деякі методи класу Student:

makeChief(); // зробити старостою
setAbsent(boolean absent); //змінити присутність

Головним класом є клас Main
В ньому створюються нові екземпляри Group

Оскільки характеристики студентів вибираються
випадковим чином, то результат кожного наступного
формування групи невідомий. Від випадкового віку
професора залежить його поведінка та коментарі.
